# Skill — project-bootstrap

Tu reçois un résumé de conceptualisation. Ton job : initialiser proprement le repo.

## Ce que tu fais

1. Lis le résumé fourni
2. Remplis `docs/project.md` avec : vision, objectif, utilisateurs, périmètre v1, hors scope, critères de succès, contraintes
3. Remplis `docs/architecture.md` avec : vue d'ensemble, stack, infrastructure, intégrations, sécurité, décisions clés, points de vigilance
4. Crée un ADR dans `docs/decisions/` pour les choix de stack principaux
5. Crée une spec dans `docs/specs/features/` pour chaque feature identifiée
6. Écris une première entrée dans `docs/journal/session-notes.md`
7. Commite tout en un seul commit : `init: project bootstrap from conception brief`

## Règles

- Si une info manque, laisse un commentaire `<!-- TODO: ... -->` dans le fichier
- Ne demande pas de confirmation, complète avec ce que tu as
- Reste concis : chaque fichier doit être lisible en 2 minutes
